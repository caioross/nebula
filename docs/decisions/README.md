# Decision records

Choices that shape the project permanently, each in its own numbered file. A record states the context, the options that were on the table, the decision, and what it costs — so future contributors argue with the reasoning instead of reopening settled ground with the same arguments.

Records are immutable once accepted. Changing a decision means writing a new record that supersedes the old one, which stays in place marked as superseded.
