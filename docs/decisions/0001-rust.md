# 0001 — Core language: Rust

Status: accepted

## Context

Nebula is a native program with a GPU-rendered interface, an embedded inference runtime, an embedded web rendering core and long-lived processes handling untrusted input. The language decision is close to irreversible — it sets the hiring pool, the dependency ecosystem, the security posture and the contributor culture for the life of the project.

Requirements that actually discriminate between candidates: predictable performance without a garbage collector stalling the render loop; memory safety while parsing hostile input (pages, messages) for years without a CVE parade; first-class interop with C libraries (inference runtimes are C/C++); credible paths to GPU work; and an ecosystem where the projects Nebula most resembles are already succeeding.

## Options considered

**C.** Total control, smallest runtime, the language of the reference inference engines. Rejected as the *primary* language: hand-written memory management across a codebase this large, parsing adversarial input, is a standing security liability that modern attackers industrialize. The project will still touch C at FFI boundaries constantly.

**C++.** The browsers are written in it, llama.cpp is written in it, talent exists. Rejected for the same memory-safety exposure as C with a much larger language to hold in one's head, and because the strongest current wave of native, GPU-first, performance-obsessed application development — the wave Nebula wants contributors from — has moved.

**Rust.** Memory safety without garbage collection, C-grade performance, serious GPU and windowing ecosystem (wgpu, winit), Servo is native to it, llama.cpp binds cleanly, and the flagship products closest to Nebula's shape (Zed, Warp, and Servo itself) demonstrate that large, fast, beautiful native applications get built and staffed in it. The borrow checker's learning curve is real and accepted; the compile times are real and accepted.

**A GC language with native bindings (Go, Kotlin, Swift).** Rejected: GC pauses against a strict frame budget, weaker stories for embedding Servo and inference runtimes, or single-platform gravity.

## Decision

Rust, for everything above the FFI line. C/C++ dependencies are linked where they are the best tool (inference kernels), always behind Rust-owned interfaces.

## Costs accepted

Slower onboarding for contributors new to Rust; compile times that must be managed with workspace discipline; occasional unsafe blocks at FFI boundaries that receive review scrutiny far above normal code.
