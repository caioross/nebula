# Documentation

The specifications that define Nebula, written before the code. Reading order for newcomers:

1. [vision.md](vision.md) — what this is and who it is for
2. [interface.md](interface.md) — the cloud and the canvas, the whole visible product
3. [architecture.md](architecture.md) — the five subsystems and the spine
4. [routing.md](routing.md) — how written input becomes intent
5. [models.md](models.md) — the engine fleet and connected providers
6. [web-engine.md](web-engine.md) — rendering and structural reading
7. [connectors.md](connectors.md) — messaging and the outside world

Lasting choices live in [decisions/](decisions). Proposals live in [rfcs/](rfcs) — start from the [template](rfcs/0000-template.md) and read [CONTRIBUTING.md](../CONTRIBUTING.md) first.

Each document carries a status line. **Draft** means open questions remain and the structure may still move; **stable** means implementation may rely on it and changes require an RFC.
